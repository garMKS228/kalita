# Мобильное приложение KALITA 📱💳

Современное кроссплатформенное мобильное приложение для хранения и управления скидочными картами. Проект разработан на фреймворке Flutter с использованием передовых практик архитектуры и управления данными.

---

## ✨ Основные возможности (Features)

* **Хранение скидочных карт:** Хранение и быстрое добавление скидочных и бонусных карт.
* **Кошельки и поиск:** Удобная сортировка карт по кошелькам (супермаркеты, одежда, кафе) и быстрый поиск нужной карты.

---

## 🛠 Технологический стек (Tech Stack)

Проект реализован с использованием следующих технологий, инструментов и библиотек:

* **Проектирование и дизайн:**
  * **Figma** — разработка UI/UX дизайна, прототипирование интерфейсов и экспорт векторных ассетов.

* **Разработка интерфейса (UI) и логика:**
  * **Flutter & Dart** — кроссплатформенный фреймворк для сборки приложения под Android и iOS.
  * **flutter_svg** — отображение векторных SVG-иконок и графических элементов интерфейса с сохранением четкости.
  * **Кастомные шрифты** — интеграция фирменных шрифтов семейства `Actay` и `ActayWide`.

* **Работа с данными и локальное хранение:**
  * **Drift (SQLite)** — реактивная локальная база данных для надежного хранения информации на устройстве и работы в офлайн-режиме.
  * **path_provider & path** — безопасное определение путей в файловой системе устройства для хранения бд.

* **Backend и облачные сервисы (Firebase):**
  * **Firebase Core** — инициализация и связь приложения с сервисами Firebase.
  * **Firebase Authentication** — безопасная авторизация и управление учетными записями пользователей.
  * **Cloud Firestore** — облачная база данных для синхронизации информации.
  * **Firebase Messaging & flutter_local_notifications** — отправка push-уведомлений и их локальный показ на устройстве.

* **Архитектура и навигация:**
  * **GoRouter** — декларативная маршрутизация и навигация между экранами приложения.
  * **Injectable** — управление зависимостями (Dependency Injection).

* **Утилиты и инструменты:**
  * **barcode_widget** — генерация штрихкодов и QR-кодов для сканирования карт на кассе.
  * **share_plus** — шеринг информации из приложения.
  * **screenshot** — создание снимков экрана программным путем.
  * **pasteboard** — взаимодействие с буфером обмена устройства.
  * **uuid** — генерация уникальных идентификаторов.

## 🚀 Быстрый старт (Getting Started)

### Требования
* Установленный Flutter SDK (версия `^3.11.0` или выше)
* Смартфон или эмулятор (Android/iOS) для запуска

### Как запустить проект на компьютере:

1. **Скачайте проект:**
   Откройте GitHub Desktop, выберите `File` -> `Clone Repository` и склонируйте этот репозиторий к себе.

2. **Откройте проект в IDE:**
   Запустите вашу среду разработки (Visual Studio Code или Android Studio) и откройте в ней папку с проектом.

3. **Установите зависимости:**
   * **В VS Code:** Нажмите на всплывающее окно `Get Packages` в правом нижнем углу или просто сохраните файл `pubspec.yaml` (`Ctrl + S`).
   * **В Android Studio:** Нажмите кнопку `Pub get` на синей панели, которая появится вверху экрана, когда вы откроете любой `.dart` файл.

4. **Запустите приложение:**
   Выберите ваш эмулятор или подключенный телефон и нажмите **F5** (в VS Code) или зеленую кнопку **Run (Shift + F10)** на верхней панели (в Android Studio).

> ⚠️ **Важно перед запуском:** В проекте используется Firebase. Убедитесь, что у вас в папке `lib/` присутствует файл `firebase_options.dart` (он скрыт в Git из-за ключей безопасности). Если его нет, скопируйте его у коллег, иначе проект выдаст ошибку при сборке.
